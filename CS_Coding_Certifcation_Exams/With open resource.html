The requirement of a try/catch block in your code depends on the outcome you desire. If you want to avoid any sort of disruption in your code execution due to some error, then wrapping the contents of your code block in a try/catch block is a good idea. Understanding exactly how the try/catch block works in Python is quite crucial for this. In Python, whenever there comes a condition where Python doesn't know what to do next, it creates an exception object. If not handled properly, this further results in the crashing of your program. Therefore, wrapping your code in a try/catch block in Python marks critical behaviour. When something wrong happens in your code inside the try block, it immediately gets passed to the catch block in Python where you can instruct Python on how to handle these exceptions.
